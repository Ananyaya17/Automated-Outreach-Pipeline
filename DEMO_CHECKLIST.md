## Features to demonstrate
- Ocean company discovery path
- Prospeo contact discovery path
- Brevo dry-run email sending path
- Data export to `exports/leads.csv`
- Async pipeline flow with `--async-run`
- Demo fallback mode when live APIs are unavailable

## Architecture explanation
- `main.py` is the CLI entrypoint.
- `pipeline/outreach_pipeline.py` orchestrates stages and supports sync/async execution.
- `services/` contains provider clients for Ocean, Prospeo, and Brevo.
- `config/settings.py` loads environment configuration and validates required keys.

## Async workflow explanation
- `run_async()` uses `AsyncOceanClient`, `AsyncProspeoClient`, and `AsyncBrevoClient`.
- Each async client is an `httpx.AsyncClient` wrapped in an async context manager.
- The async pipeline gathers results and sends emails with async Brevo calls.

## API integration explanation
- Ocean uses `POST /v3/search/companies` with `X-Api-Token`.
- Prospeo uses `GET /v1/search/contacts?domain=<domain>` with Bearer auth.
- Brevo uses `POST /v3/smtp/email` with `api-key` header.

## Error handling explanation
- Network errors and API failures are logged and do not crash the pipeline.
- Invalid or missing API keys are caught by settings validation.
- Demo mode allows a full pipeline walkthrough without real API access.
